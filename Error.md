# 먹자 골목 관리 오류/예외 처리 기록

작성일: 2026-06-06  
테스트 기준: `docs/mukja-function-tree-ko.md`

이 문서는 기능 테스트 중 확인해야 할 예상 예외와 실제 수정/처리 내역을 기록합니다.

## 테스트 기준

기능 테스트는 기능 트리의 다음 영역을 기준으로 합니다.

- 공통: 로그인, 권한 제한, 사이드바, 언어 전환, 로그아웃, 캐시 초기화
- 홈: 최신 요청 조회, 부서 필터, 체크 저장, 요청별 메모
- 요청 내역: 주 단위 목록, 상세 화면, 카테고리 표시, 체크 수정
- 요청하기: 주문 모드, 수정 모드, 재료 CRUD, 카테고리 CRUD, 일괄 변경, 드래그
- 요청하기 메시지: 요청 저장 후 부서별 메시지 생성, 복사, 카카오톡 앱 열기
- 메뉴: 보기/수정 모드, 검색, 카테고리 필터, 가격 표시, 레시피 팝업, 메뉴 CRUD
- 레시피: 보기/수정 모드, 재료/순서 CRUD, 사진, 삭제, 드래그
- 관리자: 비밀번호 CRUD, CSV Export/Import
- 데이터/API: DB CRUD, CSV 변환, localStorage fallback

## 자동 테스트 결과

| 테스트 | 목적 | 최근 결과 |
| --- | --- | --- |
| `npm run test:user-flows` | 권한별 화면 렌더링, 접근 제한, 메뉴 가격 표시, 테스트 요청 생성/삭제 | 통과 |
| `npm run test:db-consistency` | 실제 DB 요청 C/R/U/D, 메모, 체크, 홈/요청 내역/상세 동기화 | 통과 |

주의: 두 테스트는 실제 DB에 임시 요청을 만들고 삭제합니다. 동시에 실행하면 서로의 임시 요청을 잠깐 볼 수 있으므로 최종 검증은 순차 실행합니다.

## 예상 예외와 처리 방식

| 영역 | 예외 상황 | 처리 방식 |
| --- | --- | --- |
| 로그인 | 잘못된 비밀번호 입력 | 오류 메시지 표시, 입력값 선택 |
| 권한 | 권한 없는 페이지 접근 | 사이드바 비활성화, 페이지에서는 권한 없음 메시지 표시 |
| 홈 | 저장된 요청 없음 | 빈 상태 메시지 표시 |
| 홈 | 타 부서 메모 수정 시도 | 읽기 전용으로 표시 |
| 홈 | DB 최신 데이터 지연 | 브라우저 포커스/탭 복귀 시 재조회 |
| 요청 내역 | 해당 주 데이터 없음 | 빈 상태 메시지 표시 |
| 요청 내역 | 상세 ID가 없거나 삭제됨 | 목록 화면으로 돌아가 확인 |
| 요청하기 | 권한 없는 사용자가 접근 | 권한 없음 메시지 표시 |
| 요청하기 | 선택 품목 없음 | 요청 저장을 막고 빈 요청 방지 |
| 요청하기 | 품목도 메모도 없는 상태에서 요청 클릭 | 요청 내역과 메모 모두 저장하지 않음 |
| 요청하기 | 메모만 입력 후 요청 클릭 | 요청 내역 행은 만들지 않고 독립 메모만 저장, 홈에는 메모 패널만 표시 |
| 요청하기 수정 | 같은 부서에 같은 품목명 추가/수정 | 팝업으로 중복을 알리고 저장 차단 |
| 요청하기 | 요청 메시지 생성 버튼 중복 | 별도 생성 버튼을 제거하고 `요청` 버튼 저장 후 자동 생성 |
| 요청하기 | 요청 메시지 품목이 길어져 확인 어려움 | 화면 카테고리/품목 순서대로 카테고리별 라인을 만들고 `필요합니다`는 마지막 줄로 분리 |
| 요청하기 | 카톡 버튼 위치 중복 | 메시지 헤더의 카톡 버튼을 제거하고 메시지 아래 노란색 버튼만 유지 |
| 요청하기 | 보기 모드에서도 긴 목록 이동이 불편함 | 부서/카테고리 접기/펼치기를 보기 모드에도 제공 |
| 요청하기 | 수정 폼 열린 상태에서 드래그 | 편집 폼이 열려 있는 동안 품목/카테고리 이동 전체 비활성화 |
| 요청하기 수정 | 카테고리 품목이 많아 이동이 불편함 | 카테고리 접기/펼치기, 제목 클릭 포커스, 접힌 상태 액션 최소화 |
| 요청하기 수정 | 부서 안 카테고리가 많아 전체 이동이 불편함 | 부서 제목 줄에서 해당 부서 카테고리를 한 번에 접고 펼침 |
| 요청하기 수정 | 카테고리 드롭 위치가 좁거나 불명확함 | 카테고리 기준선을 표시하고 품목 행 위에서도 카테고리 드롭이 전달되도록 범위 확대 |
| 요청하기 | 체크박스 1개 선택 후 수정 모드 | 불필요한 펼침 동작 방지 |
| 재료 DB | 기존 `section` 필드 데이터 | `category`로 변환, 구형 필드 fallback 유지 |
| 재료 DB | 카테고리 영문명 누락 | `categoryEn` 필드 추가 |
| 메뉴 DB | 메뉴 카테고리 영문명 누락 | `categoryEn` 필드 추가 |
| 메뉴 DB | DoorDash 불어/영어 혼합 | 불어 제거, 한글/영문 필드 분리 |
| 메뉴 DB | 한글 메뉴명에 영어 포함 | `nameKo` 한글명으로 정리 |
| 메뉴 수정 | 같은 메뉴명 추가/수정 | 팝업으로 중복을 알리고 저장 차단 |
| 메뉴 보기 | 가격 없음 | 가격 라벨 미표시 |
| 메뉴 보기/수정 | 카테고리 안 메뉴가 많아 이동이 불편함 | 메뉴 카테고리 접기/펼치기를 보기/수정 모드에 제공 |
| 메뉴 수정 | 새 메뉴 생성 | 빈 레시피 자동 생성 |
| 메뉴/레시피 | 수정 폼 열린 상태에서 드래그 | 해당 행 드래그 비활성화 |
| 레시피 | 사진 없는 조리 순서 | 사진 버튼 비활성화 |
| 관리자 | 마지막 관리자 삭제 | 삭제 방지 |
| CSV Import | 빈 파일 또는 잘못된 CSV | Import 실패 메시지 표시 |
| CSV Import | 기존 데이터 교체 | 확인 창 후 실행 |
| DB/API | DB 연결 실패 | localStorage 보조 저장소 사용 |
| 캐시 | 이전 화면 표시 | `reset-cache.html`로 서비스워커/캐시 정리 |

## 이번 수정/처리 내역

### 1. 재료 스키마 정리

- `section` 중심이던 재료/요청 품목 분류를 `category`로 변경했습니다.
- 구형 데이터 호환을 위해 `section` fallback은 유지했습니다.
- 재료 카테고리 영어 필드 `categoryEn`을 추가했습니다.

### 2. 메뉴 데이터 정리

- DoorDash 메뉴 데이터를 메뉴 DB 스키마에 맞춰 Import했습니다.
- `Featured Items`, `Most Ordered`는 DoorDash 중복 노출 영역이라 실제 메뉴 DB에서는 제외했습니다.
- 불어/영어 혼합 설명은 영어만 남겼습니다.
- 한글 메뉴명 필드에 남아 있던 영어 항목 13개를 한글명으로 수정했습니다.
- 메뉴 보기 모드에서 메뉴명 옆 가격을 표시하도록 수정했습니다.

### 3. 드래그와 입력 충돌 방지

- 요청하기 수정 모드에서 입력 폼이 열린 행은 드래그되지 않도록 수정했습니다.
- 메뉴 수정 모드에서 입력 폼이 열린 행은 드래그되지 않도록 수정했습니다.
- 레시피 재료/조리 순서 수정 폼이 열린 행도 드래그되지 않도록 수정했습니다.

### 4. 요청하기 수정 모드 체크박스 동작

- 수정 모드에서 체크박스가 하나만 선택된 경우 불필요한 펼침 동작을 하지 않도록 수정했습니다.

### 5. 영어 UI 문구

- 영어 Request 플로팅 검색 창의 `Previous`를 `Pre`로 줄여 잘림을 방지했습니다.

### 6. 테스트 보강

- `scripts/user-flow-test.cjs`가 테스트용 요청을 직접 생성하고 삭제하도록 보강했습니다.
- 기존 요청 내역이 비어 있어도 권한별 홈/요청 내역 테스트가 안정적으로 통과하도록 수정했습니다.
- `scripts/db-consistency-test.cjs`는 실제 DB C/R/U/D와 화면 동기화를 검증합니다.
- 요청하기 테스트에서 `요청` 버튼 클릭 후 부서별 메시지 3개와 카카오톡 열기 링크 3개가 생성되는지 검증합니다.
- 요청하기 테스트에서 생성 메시지에 카테고리 라인이 포함되는지 검증합니다.
- 요청하기 테스트에서 보기 모드 부서/카테고리 접기와 펼치기를 검증합니다.
- 요청하기 테스트에서 같은 부서의 중복 품목명 저장이 팝업으로 막히는지 검증합니다.
- 메뉴 테스트에서 카테고리 접기/펼치기와 중복 메뉴명 저장 차단을 검증합니다.

### 7. 요청 메시지와 카카오톡 버튼 정리

- `요청 메시지 만들기` 별도 버튼을 제거했습니다.
- `요청` 버튼을 누르면 요청 저장 후 부서별 메시지가 자동 생성됩니다.
- 카카오톡 열기 버튼은 각 메시지 아래에 노란색 기본 버튼 스타일로 표시합니다.
- 메시지 헤더에 있던 중복 카카오톡 버튼은 제거했습니다.

### 8. 요청 메시지 문구 순서 정리

- 부서별 요청 메시지를 화면에 저장된 카테고리 순서와 품목 순서 기준으로 생성하도록 수정했습니다.
- 품목은 카테고리별 라인으로 묶어 표시합니다.
- `필요합니다` 문구는 품목 라인 뒤 마지막 줄에 표시합니다.

### 9. 중복 이름 저장 방지

- 요청하기 수정 모드에서 같은 부서에 이미 같은 품목명이 있으면 저장하지 않고 팝업을 표시합니다.
- 메뉴 수정 모드에서 이미 같은 한글 메뉴명이 있으면 저장하지 않고 팝업을 표시합니다.
- 기존 항목을 수정할 때 자기 자신은 중복 검사에서 제외합니다.

### 10. 요청하기 편집 폼 드래그 잠금 보강

- 요청하기 수정 모드에서 품목/카테고리 편집 폼이 열려 있는 동안 전체 품목/카테고리 이동을 비활성화했습니다.
- 복사/붙여넣기를 위해 입력값을 드래그할 때 주변 리스트가 이동하지 않도록 드래그 시작 이벤트도 차단했습니다.
- 자동 테스트에서 편집 폼이 열린 동안 `draggable=true` 행이 남지 않는지 확인합니다.

### 11. 요청하기 카테고리 접기/펼치기

- 요청하기 수정 모드의 카테고리 제목 줄에 접기/펼치기 버튼을 추가했습니다.
- 접힌 카테고리는 품목 목록과 추가/수정 버튼을 숨기고, 펼침 버튼과 이동 핸들만 표시합니다.
- 카테고리명을 누르면 해당 카테고리로 포커스가 이동해 긴 목록에서 위치를 확인하기 쉽습니다.

### 12. 요청하기 부서별 접기/펼치기와 이동 위치 표시

- 요청하기의 부서 제목 줄에 전체 접기/펼치기 버튼을 추가했습니다.
- 요청하기 보기 모드에서도 부서/카테고리 접기와 펼치기가 가능하도록 수정했습니다.
- 카테고리 추가는 텍스트 버튼 대신 원형 `+` 아이콘으로 정리했습니다.
- 카테고리 사이에 회색 기준선을 표시하고, 드롭 후보 위치는 노란색 선으로 표시합니다.
- 카테고리 드롭을 품목 행 위에서도 처리할 수 있도록 이벤트 전달 범위를 넓혔습니다.
- 자동 테스트에서 부서별 전체 접기/펼치기와 카테고리 추가 아이콘을 검증합니다.

### 13. 접기/펼치기 아이콘과 메뉴 카테고리 UX 통일

- 접기/펼치기 아이콘은 같은 삼각형 SVG를 회전해 사용하도록 통일했습니다.
- 원형 버튼 안에서 밑줄과 삼각형 밑변 너비를 맞추고 선 두께를 키워 가시성을 높였습니다.
- 주변 액션 아이콘과 시각 무게가 맞도록 내부 도형 폭을 키웠습니다.
- 메뉴 페이지의 카테고리 제목 줄에도 보기/수정 모드 공통 접기/펼치기 버튼을 추가했습니다.
- 메뉴 카테고리를 접으면 메뉴 목록과 추가 버튼을 숨겨 긴 목록에서 이동하기 쉽게 했습니다.
- 자동 테스트에서 메뉴 카테고리 접기/펼치기를 검증합니다.

### 14. 빈 요청 저장 차단과 메모 전용 저장

- 요청하기에서 품목과 메모가 모두 비어 있으면 요청 내역 DB에 저장하지 않습니다.
- 메모만 입력한 경우 `orders`/`order_items` 요청 내역을 만들지 않고 `app_settings.standaloneMemos`에 독립 메모로 저장합니다.
- 홈 화면은 독립 메모가 최신이면 품목 목록 없이 메모 패널만 표시합니다.
- 자동 테스트에서 빈 요청 저장 차단과 메모만 저장 시 요청 내역이 늘지 않는지 검증합니다.
