# 먹자 골목 관리 사용자 매뉴얼

이 문서는 먹자 골목 관리 앱을 실제 사용자에게 설명하기 위한 한글 매뉴얼입니다.

접속 주소:

```text
https://mukjamtl.netlify.app
```

## 1. 로그인

앱은 회원가입 없이 입장 비밀번호로 권한을 구분합니다.

| 사용자 | 비밀번호 | 사용 가능한 주요 기능 |
| --- | --- | --- |
| 카페테리아 | `c1234` | 카페테리아 요청 확인, 메모 작성 |
| 야채 | `v1234` | 야채 요청 확인, 메모 작성 |
| 그로서리 | `g1234` | 그로서리 요청 확인, 메모 작성 |
| 레스토랑 | `m1234` | 요청 작성, 전체 요청 확인, 입고 확인, 메뉴/레시피 조회 |
| 관리자 | `madmin` | 전체 기능, 재료/메뉴/레시피/비밀번호/CSV 관리 |

휴대폰에서는 브라우저의 비밀번호 저장 기능을 사용하면 다음 접속부터 더 빠르게 로그인할 수 있습니다.

## 2. 사이드바 메뉴

왼쪽 사이드바에서 페이지를 이동합니다.

![사이드바 메뉴](docs/images/manual-sidebar.png)

| 아이콘 | 메뉴 | 설명 |
| --- | --- | --- |
| 집 | 홈 | 최근 요청과 입고/메모 확인 |
| 시계 | 요청 내역 | 저장된 요청 내역 조회 |
| 요청 | 요청하기 | 레스토랑/관리자 요청 작성 |
| 메뉴 | 메뉴 | 메뉴와 레시피 조회/관리 |
| 관리자 | 관리자 | 비밀번호와 CSV 관리 |

현재 페이지는 노란색 원 아이콘으로 표시됩니다. 권한이 없는 메뉴는 흐리게 표시되고 사용할 수 없습니다.

## 3. 홈

홈은 로그인한 사용자에 따라 보이는 내용이 다릅니다.

![홈 화면](docs/images/manual-home.png)

카페테리아, 야채, 그로서리 사용자는 자기 부서로 들어온 최근 요청만 봅니다.

레스토랑과 관리자는 전체 부서의 최근 요청을 봅니다. 레스토랑 사용자는 품목 오른쪽 체크 박스로 입고 확인을 저장할 수 있습니다.

메모는 요청별로 저장됩니다. 다른 부서의 메모는 읽기 전용으로 보이고, 내 부서 메모만 수정할 수 있습니다. 메모가 없으면 `메모 없음`으로 표시됩니다.

## 4. 요청 내역

요청 내역에서는 저장된 요청을 주 단위로 확인합니다.

![요청 내역](docs/images/manual-history.png)

요청 내역은 화요일부터 일요일까지 한 주 기준으로 묶입니다. 아래 페이지 버튼으로 이전/다음 묶음을 이동합니다.

리스트에서 행을 누르면 상세 내역이 열립니다.

![상세 내역](docs/images/manual-history-detail.png)

상세 내역은 부서별, 카테고리별로 표시됩니다. 관리자는 출고 확인과 입고 확인 체크를 수정할 수 있고, 일반 부서 사용자는 읽기 중심으로 확인합니다.

닫기 버튼을 누르면 요청 내역으로 돌아갑니다.

## 5. 요청하기

요청하기는 레스토랑과 관리자만 사용할 수 있습니다.

![요청하기](docs/images/manual-request.png)

주문 모드에서는 필요한 품목을 체크하고 메모를 작성한 뒤 `요청` 버튼을 누릅니다.

요청을 저장하면 새로운 요청 내역으로 DB에 저장됩니다. 같은 날 여러 번 요청을 저장해도 이전 요청을 덮어쓰지 않고 요청 내역에 각각 남습니다.

관리자는 수정 모드에서 재료 품목과 카테고리를 관리할 수 있습니다. 여러 품목을 선택한 뒤 부서/카테고리를 한 번에 변경하거나, 순서를 이동할 수 있습니다.

## 6. 메뉴

메뉴 화면은 레스토랑과 관리자만 사용할 수 있습니다.

![메뉴](docs/images/manual-menu.png)

카테고리와 검색어로 메뉴를 찾을 수 있습니다. 판매 중단 메뉴는 메뉴명에 취소선으로 표시됩니다.

레시피 아이콘을 누르면 해당 메뉴의 레시피가 열립니다. 관리자는 수정 모드에서 메뉴를 추가, 수정, 삭제할 수 있습니다.

## 7. 레시피

레시피는 메뉴와 연결되어 있습니다.

![레시피](docs/images/manual-recipe.png)

보기 모드에서는 재료와 조리 순서를 확인합니다. 수정 모드에서는 관리자가 재료와 조리 순서를 추가, 수정, 삭제하고 순서를 변경할 수 있습니다.

레시피 삭제는 확인 창을 거쳐 진행됩니다.

## 8. 관리자

관리자 페이지는 관리자만 사용할 수 있습니다.

![관리자](docs/images/manual-admin.png)

비밀번호 관리에서는 입장 비밀번호와 권한을 관리합니다. 사용자 추가는 제목 오른쪽 `+` 버튼으로 진행하고, 삭제는 수정 폼 안에서만 할 수 있습니다.

CSV 관리는 네 가지 데이터를 가져오거나 내보냅니다.

| 항목 | 설명 |
| --- | --- |
| 요청 내역 | 저장된 요청과 입고 확인 데이터 |
| 재료 목록 | 요청하기 화면에서 사용하는 재료 품목 |
| 메뉴 | 메뉴명, 카테고리, 판매 상태 등 |
| 레시피 | 재료, 조리 순서, 메모 등 |

CSV 가져오기는 기존 데이터를 바꿀 수 있으므로 확인 창을 거친 뒤 실행됩니다.

## 9. 화면이 예전처럼 보일 때

아이폰 사파리나 모바일 브라우저에서는 캐시 때문에 이전 화면이 잠시 남을 수 있습니다.

아래 주소를 한 번 열면 캐시를 비우고 앱으로 돌아갑니다.

```text
https://mukjamtl.netlify.app/reset-cache.html
```

그래도 화면이 바뀌지 않으면 브라우저 탭을 완전히 닫고 다시 열어 주세요.
