# 먹자 골목 관리

먹자 골목 관리는 식당 요청 확인, 입고 현황 관리, 메뉴/레시피 확인, 요청 내역 관리, CSV 가져오기/내보내기를 모바일과 데스크톱 브라우저에서 빠르게 처리하기 위한 웹앱입니다.

현재 버전은 **간편모드 중심**으로 개발 중입니다. 사용자가 별도로 일반모드 개발을 요청하기 전까지 화면과 기능은 간편모드 기준으로 정리합니다.

## 접속 주소

Netlify 배포 주소:

```text
https://mukjamtl.netlify.app
```

로컬 확인 주소:

```text
http://127.0.0.1:4173
```

## 주요 기능

- 홈에서 부서별 받은 요청 또는 전체 입고 현황 확인
- 주문 부서별 재료 주문 작성
- 카페테리아 품목을 섹션별로 확인
- 요청 내역 주간 조회 및 상세 확인
- 메뉴 검색, 카테고리별 메뉴 조회, 레시피 연결
- 레시피 조회
- 관리자 페이지에서 비밀번호, 재료, 섹션, 메뉴, 레시피, CSV 관리
- Netlify DB 연결 시 주문/입고 내역 서버 저장
- 입장 비밀번호에 따른 역할/부서 제한

## 입장 비밀번호

현재 버전은 회원가입 없이 입장 비밀번호로 권한을 구분합니다.

| 역할 | 비밀번호 | 권한 |
| --- | --- | --- |
| 카페테리아 | `c1234` | 카페테리아 부서 주문 화면 |
| 야채 | `v1234` | 야채 부서 주문 화면 |
| 그로서리 | `g1234` | 그로서리 부서 주문 화면 |
| 레스토랑 | `m1234` | 전체 확인 및 입고 저장 |
| 관리자 | `madmin` | 전체 관리 기능 |

모바일 브라우저의 비밀번호 저장 기능을 사용하면 반복 로그인 부담을 줄일 수 있습니다.

## 화면 구성

### 홈

홈에서는 로그인한 역할에 따라 다른 정보를 보여줍니다.

카페테리아, 야채, 그로서리 담당자는 자기 부서로 들어온 최근 요청을 확인합니다.

레스토랑과 관리자는 전체 부서의 입고 현황을 확인하고 입고 체크를 저장할 수 있습니다.

### 주문

주문 화면에서는 `주문 부서`를 선택한 뒤 필요한 품목을 체크합니다.

부서 비밀번호로 입장한 경우 해당 부서만 표시됩니다. 레스토랑과 관리자는 전체 부서를 볼 수 있습니다.

현재 주문 부서는 다음 세 가지입니다.

- 카페테리아
- 야채
- 그로서리

카페테리아 품목은 아래 섹션으로 나누어 표시됩니다.

- 반조리
- 반찬
- 소스
- 냉장
- 냉동

간편모드에서는 품목 행에 **항목명만 표시**합니다.

### 요청 메시지

품목을 선택하거나 메모를 입력한 뒤 `저장`을 누르면 요청 리스트와 메모가 저장됩니다.

메모는 작성 역할과 부서별로 구분됩니다.

부서 담당자는 자기 부서 메모와 레스토랑/관리자 메모를 볼 수 있고, 레스토랑과 관리자는 모든 부서의 메모를 확인할 수 있습니다.

### 요청 내역

저장된 요청 내역을 화요일부터 일요일까지 주 단위로 확인할 수 있습니다.

페이지 이동은 `<<`, `<`, 주차 번호 5개, `>`, `>>` 방식으로 표시됩니다.

목록의 각 행을 선택하면 요청 세부 내역과 메모를 확인할 수 있고, `닫기`를 누르면 요청 내역 목록으로 돌아갑니다.

간편모드 요청 내역은 품목명 목록 중심으로 표시됩니다.

### 메뉴

메뉴 화면은 레스토랑과 관리자만 사용할 수 있습니다.

현재 판매 중인 메뉴와 판매 중단 메뉴를 모두 카테고리별로 보여주며, 판매 중단 메뉴는 흐리게 표시됩니다.

메뉴 검색을 사용할 수 있고, 메뉴 상세에서 연결된 레시피로 이동할 수 있습니다.

### 레시피

등록된 레시피를 검색하고 상세 내용을 확인할 수 있습니다.

메뉴에서 레시피로 들어온 경우 `닫기`를 누르면 메뉴 화면으로 돌아갑니다.

### 관리자

관리자 페이지에서는 다음 데이터를 관리할 수 있습니다.

- 부서/역할 비밀번호
- 섹션
- 재료
- 메뉴
- 레시피
- 주문 리스트 CSV
- 레시피 CSV
- 전체 설정 JSON
- 테스트 데이터 생성
- DB 초기화

메뉴와 레시피는 재판매와 재사용 가능성을 고려해 삭제하지 않고 사용/중지 상태로 관리합니다.

### 테스트 데이터

관리자 페이지의 `테스트 데이터 생성` 버튼을 누르면 테스트용 재료, 메뉴, 레시피, 요청 내역이 넉넉히 생성됩니다.

요청 내역은 최근 10주 동안 화요일부터 일요일까지 생성되어 홈, 요청 내역, 상세 화면, 페이지네이터를 테스트할 수 있습니다.

`DB 초기화` 버튼을 누르면 요청 내역을 비우고 재료, 메뉴, 레시피, 입장 비밀번호를 기본값으로 되돌립니다.

DB 환경변수가 아직 연결되지 않은 배포 환경에서는 새 브라우저가 처음 열릴 때 테스트 데이터가 localStorage에 자동 생성됩니다. 이 경우 데이터는 브라우저별로 따로 보입니다.

## 배포 방식

이 프로젝트는 정적 웹앱과 Netlify Functions를 함께 사용합니다.

Netlify 설정은 `netlify.toml`에 들어 있으며, 현재 publish directory는 저장소 루트입니다.

```toml
[build]
  publish = "."
  command = ""

[functions]
  directory = "netlify/functions"
```

GitHub `main` 브랜치에 변경 사항을 push하면 Netlify가 자동으로 배포합니다.

## 데이터 저장 방식

현재 버전은 **DB 우선 저장 + localStorage 보조 저장** 구조입니다.

Netlify DB 환경변수가 준비된 배포 환경에서는 주문, 입고 확인, 레시피가 DB에 저장됩니다.

DB 연결이 없거나 로컬 서버처럼 Functions가 실행되지 않는 환경에서는 기존처럼 브라우저 `localStorage`에 저장됩니다. 따라서 개발 중에는 화면 테스트가 끊기지 않습니다.

현재 DB는 PostgreSQL 기준이며, 아래 10개 테이블을 사용합니다.

| 테이블 | 목적 |
| --- | --- |
| `access_identities` | 브라우저별 익명 사용자, 역할, 부서 기록 |
| `access_logs` | API 접속 IP, 기기, 역할 로그 |
| `access_accounts` | 입장 비밀번호, 역할, 부서, 표시 이름 |
| `orders` | 주문 날짜, 시간, 메모, 요청 메시지 |
| `order_items` | 주문별 품목, 섹션, 부서, 입고 상태 |
| `order_memos` | 역할/부서별 주문 메모 |
| `receipt_confirmations` | 우리 쪽 입고 확인 기록 |
| `department_confirmations` | 추후 부서별 출고 확인 기록 |
| `recipes` | 레시피 본문, 재료, 순서, 메모, 이미지 URL |
| `menus` | 메뉴 한글/영문 이름, 카테고리, 계절 메뉴, 판매 중단, 가격, 연결 레시피 |

테스트 데이터 생성과 초기화는 `/api/seed-data`를 통해 위 테이블의 운영 데이터를 테스트 묶음 또는 기본 묶음으로 교체합니다. 메뉴와 레시피 일반 관리 화면에는 삭제 기능을 제공하지 않습니다.

### 메뉴 DB 스키마

`menus` 테이블은 판매 메뉴 정보를 저장하고, `recipe_id`로 `recipes.id`와 연결합니다.

| 컬럼 | 타입 | 설명 |
| --- | --- | --- |
| `id` | `text primary key` | 메뉴 ID |
| `recipe_id` | `text` | 연결된 레시피 ID |
| `category` | `text` | 메뉴 카테고리 |
| `name_ko` | `text` | 메뉴 한글 이름 |
| `name_en` | `text` | 메뉴 영어 이름 |
| `seasonal` | `boolean` | 계절 메뉴 여부 |
| `discontinued` | `boolean` | 판매 중단 여부 |
| `price` | `numeric(10, 2)` | 가격 |
| `currency` | `text` | 통화, 기본값 `CAD` |
| `notes` | `text` | 메뉴 메모 |
| `changed_by_identity_id` | `text` | 마지막 수정자 |
| `changed_ip` | `text` | 마지막 수정 IP |
| `changed_user_agent` | `text` | 마지막 수정 기기 정보 |
| `created_at` | `timestamptz` | 생성 시간 |
| `updated_at` | `timestamptz` | 수정 시간 |

메뉴와 레시피는 삭제하지 않습니다. 메뉴는 `discontinued`로 판매 중단 처리하고, 레시피는 `enabled`로 사용/중지를 관리합니다.

접속 IP와 브라우저 정보는 화면에 표시하지 않고, 나중에 로그인/회원 기능을 붙일 때 감사 기록과 비교 기능을 만들기 위한 서버 기록으로만 사용합니다.

주문 메모는 작성 역할과 부서별로 분리해 저장합니다. 레스토랑/관리자는 모든 메모를 볼 수 있고, 부서 담당자는 자기 부서 메모와 레스토랑/관리자 메모만 볼 수 있습니다.

레시피는 관리자 화면에서 CSV 가져오기/수정/삭제 시 DB에 동기화됩니다. DB 연결이 없으면 기존처럼 브라우저 로컬 저장소만 사용합니다.

DB 연결에 사용하는 값은 저장소에 직접 넣지 않습니다. Netlify UI에서 Functions scope 환경변수로 설정해야 합니다.

지원하는 환경변수 이름은 아래 순서로 확인합니다.

- `NETLIFY_DATABASE_URL`
- `NETLIFY_DATABASE_URL_UNPOOLED`
- `DATABASE_URL`
- `POSTGRES_URL`
- `NEON_DATABASE_URL`

DB 상태 확인 API:

```text
https://mukjamtl.netlify.app/api/health
```
